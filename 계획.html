📋 프로젝트 계획서: Simple Board Prototype

1. 기술 스택

HTML/CSS: 기본 웹 구조 및 스타일링
TypeScript: 안정성을 높인 JavaScript 슈퍼셋
Node.js (Express): 백엔드 서버 런타임 및 프레임워크
Neon (PostgreSQL): 서버리스 관계형 DB 서비스

2. 시각적 디자인 샘플 (5개)

1. 미니멀 화이트 (Minimal White)

흰색 배경과 연한 회색 테두리, 높은 여백을 활용한 극도로 깔끔한 디자인.

2. 다크 모던 (Dark Modern)

어두운 네이비/그레이 배경에 밝은 텍스트, 현대적인 느낌의 세련된 디자인.

3. 클래식 포럼 (Classic Forum)

정갈한 표 형식의 목록과 전통적인 웹 게시판 느낌의 디자인.

4. 소프트 블루 (Soft Blue)

파스텔 톤의 파란색을 메인으로 사용하여 눈이 편안하고 친근한 디자인.

5. 네이처 그린 (Nature Green)

초록색 포인트 컬러와 부드러운 곡선을 강조한 생동감 있는 디자인.

3. 데이터베이스 설계 (ERD 및 Create 문)

테이블 목록

테이블명설명
users회원 정보 (아이디, 비밀번호 등)
posts게시글 정보 (제목, 내용, 작성자, 작성일)
comments댓글 정보 (내용, 작성자, 게시글 연결)
likes추천 정보 (계정당 1회 제한용)

전체 Create 문

CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE comments (
    id SERIAL PRIMARY KEY,
    post_id INTEGER REFERENCES posts(id) ON DELETE CASCADE,
    user_id INTEGER REFERENCES users(id),
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE likes (
    id SERIAL PRIMARY KEY,
    post_id INTEGER REFERENCES posts(id) ON DELETE CASCADE,
    user_id INTEGER REFERENCES users(id),
    UNIQUE(post_id, user_id)
);
